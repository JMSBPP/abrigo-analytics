## Phase 2 — Implementation

### Phase 2 prelude — Behavior contracts: math pins (CORRECTIONS-α §15.2)

> Resolves MQ-BLOCK-1, MQ-BLOCK-2, MQ-BLOCK-3, MQ-BLOCK-4 from v0.1 reverify. These pins are **inputs to every Phase-2 task agent dispatch**. The implementing agent reads these contracts before authoring code; foreground enforces by including the relevant pin in each Task 2.x dispatch brief.

**Pin M1 — TruncPareto $\alpha$-floor (spec §8(6); resolves MQ-BLOCK-1).** The TruncPareto sampler in `simulations/modules/` must accept $\alpha \ge 1.5$ and refuse $\alpha < 1.5$ at sampler-construction time (raise a typed `ValueError`). Saas-specific commitment per spec §8(6); this is a **per-cohort guard** the Callable surfaces, NOT a generic distributional invariant on `simulations/types/TruncParetoParams` itself (resolves MQ-FLAG saas-α-floor-leak: `simulations/types/TruncParetoParams` accepts any $\alpha > 0$; the floor binds at the sampler boundary). The hypothesis test in 3.3 must include a strategy that draws $\alpha < 1.5$ and asserts the sampler raises.

**Pin M2 — Softplus $\beta$-tightness (spec §5.1 (T2); resolves MQ-BLOCK-2).** The softplus regularizer $\mathrm{softplus}_\beta(x) = \beta^{-1}\log(1 + e^{\beta x})$ must be parameterized such that the $L^1$ deviation from $(\cdot)^+$ over the support $[0, 2\kappa]$ is $< 10^{-3} \cdot \kappa$. `SoftplusParams` must encode $(\beta, \kappa)$ and a `tightness_l1_deviation()` method (or free function) that returns the computed deviation; `SoftplusRegularizer` Callable refuses $\beta$ values that fail this criterion.

**Pin M3 — Blended $p_t$ formula (spec §5.1; resolves MQ-BLOCK-3 and RC-FLAG-4 from v1.0 spec).** The blended-price function in `simulations/modules/` MUST implement *verbatim*:

$$p_t \;=\; w_{\text{in}} \cdot p_{\text{in}} \cdot (1 - h_{\text{cache}} + h_{\text{cache}} \cdot 0.10) \;+\; w_{\text{out}} \cdot p_{\text{out}}$$

where $w_{\text{in}} = 0.539$, $w_{\text{out}} = 0.461$ (arxiv 2601.14470 input-dominance), $h_{\text{cache}}$ is the cache-hit fraction (default 0.95 per spec §5.2), and $(p_{\text{in}}, p_{\text{out}})$ are tier-dependent per spec §5.2 pricing table (Sonnet 4.6 $3/$15; Opus 4.6 $5/$25; Haiku 4.5 $1/$5). For Sonnet at default cache, this evaluates to $\approx 7.15$/MTok (verified in spec v1.1.1 §5.4 footnote). The function returns $\$/$MTok; dimensional annotation per the `dimensional-analysis` skill is required on the return type.

**Pin M4 — Parquet column schemas (spec §10; resolves MQ-BLOCK-4).** The IO boundary in `simulations/utils/` MUST emit the following columns *verbatim* per spec §10:

- `cohort_prior.parquet`: `param`, `percentile`, `value`, `source`, `fetched_at_utc`.
- `synthetic_tau_t.parquet`: `month`, `simulation_id`, `tier_id`, `r`, `p`, `alpha`, `x_m`, `tau_t`, `q_t_usd`, `q_t_cop`.
- `Z_cap_pinned.json` (TODO-COHORT-4 emission, but schema fixed here): `Z_cop_per_month`, `ci_95_lo`, `ci_95_hi`, `audit_block`, `tier_mix` (object).

The IO boundary class signature must accept a typed Value object (not a `dict[str, Any]`) and validate column presence on read.

**Pin M5 — FX path equation source anchor (resolves MQ-FLAG: FX-path source).** The FX path Callable in `simulations/modules/` implements PRIMITIVES.md (6) verbatim:

$$(X/Y)_t \;=\; \big(1 + \varepsilon\,(\cos^2(\omega t) - \tfrac{1}{2})\big)\,\overline{X/Y},\quad 0 < \varepsilon < 1.$$

The variance inversion $\varepsilon(\sigma_T) = \sqrt{8\sigma_T / \overline{X/Y}^2}$ is PRIMITIVES.md (8). Both equation references MUST appear in the Callable's docstring.
