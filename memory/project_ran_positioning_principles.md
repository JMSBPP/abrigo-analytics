Founder's positioning principles for all branding, pitch, and product-surface decisions:

1. **Target = non-crypto mainstream users.** Crypto is under the hood. Users (households, remitters, small businesses in underserved-FX countries) must never need to know they are using crypto to receive value. Institutions may get a different interface with more technical surface, but the retail path is fully abstracted.

2. **Never sell decentralization.** Mainstream users do not care about decentralization. Selling it is a losing move. Sell the hedge, the protection, the preserved purchasing power.

3. **Painkiller, not vitamin.** The product must be a must-have for a real, felt problem (inflation eating savings, remittance value evaporating between send and spend, devaluation wiping out income). Not a nice-to-have. If it feels like a "vitamin," the framing is wrong.

4. **10x better than existing alternatives.** The comparison set is traditional hedges available to people in underserved-FX markets: USD savings accounts, informal dollarization, physical-asset hoarding, remittance services. The pitch must make the improvement over these concrete and felt.

5. **Education is friction. Friction kills adoption.** If the product requires teaching the user what a "pool" or "note" or "option" is, the framing has failed. Wrap the mechanics in language the user already has (e.g., "protect," "save," "backed").

6. **Two segment posture:** crypto-native users are a different market with different rules. We are NOT building for them. If an artifact would read naturally only to a crypto-native audience, it is off-positioning.

**Why:** founder has ~10 years building traditional startups with real paying customers, 1 year in crypto, and has watched crypto products fail by selling decentralization to users who wanted a solved problem.

**How to apply:** every branding artifact, name choice, hackathon application answer, one-pager, and public-facing copy decision runs through these six principles. Internal engineering docs are exempt \u2014 specs and plans continue to use precise technical language.
